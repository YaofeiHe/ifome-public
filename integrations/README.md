# Integrations

这里预留外部连接器和通知适配器目录。

约束：

- 连接器只负责和外部系统交互，不写业务判断。
- 任何第三方 SDK 都不要直接散落到 `core/` 或 UI 里。
- 失败与重试策略需要通过统一接口暴露给主链路。

目录命名原则：

- 连接器名称应尽量对应具体平台，而不是模糊业务名。
- 例如 `boss_zhipin/` 明确表示 Boss 直聘，而不是泛化成 `boss/`。

`boss_zhipin/` 的职责：

- 读取 Boss 直聘岗位详情。
- 读取 Boss 直聘 HR 对话内容。
- 把平台数据转换成内部统一输入。
- 当前最小实现包含 connector contracts 和 translator，不直接依赖真实平台抓取。

`boss_zhipin/` 不负责：

- 岗位匹配和优先级打分。
- 面试建议或聊天回复策略生成。
- 默认自动代替用户发消息。
