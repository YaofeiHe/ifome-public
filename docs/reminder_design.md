# Reminder Design

## 目标

提醒系统的任务不是直接发通知，而是先把重要事件转成可调度、可追踪、可重试的提醒任务。

## 最小实体

Reminder Queue 中的最小单位是 `ReminderTask`，而不是某个裸字段。一个提醒任务至少应包含：

- 来源对象 ID
- 用户与工作区范围
- 标题和摘要
- 触发时间
- 优先级
- 渠道
- 状态
- 调度 payload

## 第一阶段范围

- 从岗位 DDL、笔试、面试、宣讲等时间字段生成提醒任务。
- 先落 SQLite 队列，不直接发送真实通知。
- 允许一个事件生成多个提醒时间点，例如 T-24h 和 T-2h。

## 后续扩展

以下能力现在只预留接口，不进入主实现：

- 邮件提醒
- 日历事件
- Web Push

当前代码约束：

- 提醒任务通过 `ReminderQueue` 写入本地 SQLite 队列
- 读取接口直接从提醒队列表读取
- 邮件 / 日历 / Web Push 仅保留 preview adapter，不做真实发送

## 设计约束

- 时间字段统一为标准 datetime 格式。
- 时间模糊时要显式标记置信度，不能伪造精确时间。
- 提醒创建与提醒发送解耦，便于测试和回放。
